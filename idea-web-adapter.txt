IoT MQTT Web Adapter

DB:
thing:
- id
- class (leds, powerswitch, ...)
- device-id (esp-xxxxxx)

state:
- id
- thing-id
- event-source (local, mqtt)
- when
- status-string


MQTT-Adapter:
Subscribed pro thing auf /$class/$device-id/state
Speichert Nachrichten in der DB
Kann via Schnittstelle (HTTP?) Zeug nach /$class/($device-id/)action schreiben


Webseite:
Liest Status (regelmäßig) aus DB
Weiß, was in welches topic für welche aktion geschrieben werden muss
redet (über Schnittstelle) mit MQTT-Adapter
